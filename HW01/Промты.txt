#1
Напиши промпт для выолнения задания
необходимо сгенерировать и собрать простое приложение «Мини-анкета», состоящее из backend-части и небольшого frontend-интерфейса.

Требуемый функционал
Backend предоставляет два API:

GET /questions — возвращает список вопросов анкеты (жёстко заданных, 3–5 шт.).
POST /answers — принимает ответы пользователя и сохраняет их в памяти (например, в slice/array/map).
Frontend:

загружает вопросы с backend;
отображает их в HTML/JS интерфейсе;
отправляет заполненные ответы через POST /answers;
показывает пользователю сообщение «Спасибо!» после отправки.
Технологии (на выбор студента)
Backend: Python, (Допустимо Node.js/TS, Go)
Frontend: чистый HTML+JS, небольшой React/Vue или шаблон

#2
Ты — опытный fullstack-разработчик. Твоя задача — создать простое веб-приложение «Мини-анкета» на Python (backend) и HTML/CSS/JS (frontend). Используй следующие технологии:

- Backend: Python + FastAPI (или Flask, на твоё усмотрение). Сервер должен запускаться на локальном хосте, порт 8000.
- Frontend: один HTML-файл со встроенными стилями и JavaScript.

Требования к функционалу:

1. Backend предоставляет два REST API:
   - GET /questions — возвращает JSON-массив из 3–5 вопросов. Вопросы могут быть любые (например, "Ваше имя?", "Ваш возраст?", "Ваш город?" и т.д.). Жёстко заданы в коде (не из базы данных).
   - POST /answers — принимает JSON в формате { "answers": [ {"question": "...", "answer": "..."}, ... ] } или просто массив ответов в том же порядке, что и вопросы. Сохраняет ответы в памяти сервера (в список/словарь/массив). После сохранения возвращает статус 200 и сообщение { "message": "Спасибо!" }.

2. Frontend:
   - При загрузке страницы отправляет GET-запрос на /questions и получает список вопросов.
   - Динамически отображает каждый вопрос вместе с полем ввода (текстовое поле, текстареа или radio/checkbox — на твоё усмотрение, но лучше текстовые поля для простоты).
   - Под вопросами есть кнопка «Отправить».
   - При нажатии на кнопку собирает ответы (вопрос + ответ пользователя) и отправляет POST-запрос на /answers.
   - После успешной отправки показывает пользователю сообщение «Спасибо!» (можно в виде alert или отдельным блоком на странице).
   - Обрабатывает ошибки сети или сервера (показывает сообщение об ошибке).

3. Общие требования:
   - Код должен быть полностью самодостаточным. Предоставь один файл backend (например, main.py) и один файл frontend (index.html) или объедини frontend как статику внутри backend.
   - Добавь инструкцию по запуску (какие библиотеки установить, как запустить сервер, как открыть фронтенд).
   - Для локальной разработки настрой CORS, чтобы фронтенд (например, открытый из файла или на другом порту) мог обращаться к backend.
   - Хранение ответов — в памяти сервера (простой список или словарь). При перезапуске сервера данные сбрасываются — это нормально.

Выдай итоговый код в виде сообщения с пояснениями и файлами.
